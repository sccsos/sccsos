## 二、安全组件接线审计

### 2.1 安全模块文件清单

| 文件 | 功能 | 接线状态 |
|------|------|---------|
| `sccsos/security/base.py` | 抽象基类 (PolicyEngineABC, SandboxABC, 异常类) | 被 policy.py 和 sandbox.py 继承 |
| `sccsos/security/policy.py` | 策略引擎 (预算/工具权限) | ✅ 已接线 |
| `sccsos/security/sandbox.py` | 命令沙箱 (白名单/危险模式) | ✅ 已接线 |
| `sccsos/security/injection.py` | 提示注入检测 (多语言/重写检测) | ❌ 未接线 |
| `sccsos/security/rbac.py` | API 层 RBAC (权限/角色) | ✅ 已接线 |
| `sccsos/security/ratelimit.py` | 令牌桶限流器 | ❌ 未接线 |
| `sccsos/security/__init__.py` | 模块入口 (仅注释) | ✅ |

### 2.2 已接线组件详情

#### ✅ PolicyEngine (`policy.py`)
- **实例化位置**: `WorkflowRuntime.initialize()` → `PolicyEngine(core.db, cfg)`
- **注入位置**: WorkflowEngineBuilder → `.with_policy_engine(policy_engine)`
- **运行时引用**: `AgentRuntime.policy_engine` 属性 → `self._wf.engine._policy_engine`
- **调用点**: `AgentRuntime.register_agent()` 中检查 `check_agent_toolsets()` + 抛出 `PolicyViolation`
- 异常处理: `try/except` 包裹，失败时记录 CRITICAL 日志但**不阻止启动**

#### ✅ CommandWhitelist (`sandbox.py`)
- **实例化位置**: `RuntimeCore.initialize()` → `CommandWhitelist(allowed_commands=..., dangerous_patterns=...)`
- **注入位置**: `create_adapter(mode=..., whitelist=sandbox)` → `HermesSubprocessAdapter`
- **调用点**: `HermesSubprocessAdapter`.`_run_command()` / `_run_streaming()` / `run_with_config()` — 每次 shell 调用前执行 `_sandbox_check()`
- 危险模式从 `cfg.policies.default.dangerous_patterns` 读取

#### ✅ RBAC (`rbac.py`)
- **实例化位置**: 模块级单例 `_checker = RoleChecker()`
- **使用方式**: FastAPI `Depends(require_permission(...))` 在路由注册时绑定
- **覆盖路由**:
  - `routes/agents.py` — agents:read, agents:manage
  - `routes/billing.py` — billing:read, billing:export
  - `routes/quotas.py` — quota:read, quota:write
  - `routes/sessions.py` — sessions:read, sessions:write
  - `routes/skills.py` — skills:approve
  - `routes/traces.py` — traces:read
  - `routes/webhooks.py` — webhooks:read, webhooks:write
  - `routes/workflows.py` — workflows:read, workflows:write
- 角色层级: admin > operator > viewer; 默认角色: `viewer` (X-Role 头)

### 2.3 未接线组件详情

#### ❌ PromptInjectionGuard (`injection.py`)
- **代码状态**: 功能完整，含 NFKC Unicode 标准化、Cyrillic 混淆字母转换、多语言注入模式（中/日/韩/法）、SQL/XSS/命令注入检测、敏感数据自动脱敏
- **预期接线点**（推测）:
  - API 路由输入中间件 — 检查用户输入提示是否包含注入
  - HermesAdapter 消息发送前 — 检查发送给 agent 的提示是否安全
- **当前**: 无 import、无实例化、无调用

#### ❌ RateLimiter (`ratelimit.py`)
- **代码状态**: 功能完整，令牌桶算法，支持 per-key 隔离、周期性清理、burst 容量、线程安全
- **预期接线点**（推测）:
  - API 路由中间件 — 按 tenant/agent/IP 限流
  - HermesAdapter 请求限流
- **当前**: 无 import、无实例化、无调用

## 三、问题摘要

### 🔴 P0 — 构建阻断

| # | 问题 | 涉及文件 | 影响 |
|---|------|---------|------|
| 1 | Dockerfile 版本标签/引用全部为 `0.15.5` | `Dockerfile`, `Dockerfile.hermes`, `Dockerfile.slim` | 构建 LABEL 错误；pip install 硬编码 wheel 名与当前版本不匹配 → **镜像构建可能失败** |

### 🟡 P1 — 安全盲区

| # | 问题 | 涉及文件 | 影响 |
|---|------|---------|------|
| 2 | PromptInjectionGuard 已实现但未接线 | `injection.py` | 提示注入攻击无保护 |
| 3 | RateLimiter 已实现但未接线 | `ratelimit.py` | API/agent 调用无速率限制，存在 DoS 风险 |

### 🩶 P2 — 建议优化

| # | 问题 | 涉及文件 | 影响 |
|---|------|---------|------|
| 4 | `deploy/k8s/40-service.yaml` 缺少 `app.kubernetes.io/version` label | `40-service.yaml` | 不一致，但不影响功能 |

## 四、建议修复顺序

1. **Dockerfile 版本同步** — 将三个 Dockerfile 中的 `0.15.5` 全部更新为 `0.16.5`
2. **PromptInjectionGuard 接线** — 在 `HermesAdapter` 的消息发送流程中插入 `PromptInjectionGuard.check()`
3. **RateLimiter 接线** — 在 FastAPI app 注册中间件或路由依赖中集成 `RateLimiter`
4. **K8s 标签补全** — 为 service/HPA 添加 `app.kubernetes.io/version` label
